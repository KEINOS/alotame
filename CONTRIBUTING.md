# Contributing Guide for Alotame

## Pull Requests

Any PR for the betterment of Alotame is welcome!

- Branch to PR:
  - `main`
- No massive changes in a single PR
  - I am single threaded, so smaller PRs are easier to review and merge
- Requirements to merge:
  - Passing CI checks
  - At least one approving review or approval from maintainer
  - No unresolved comments (TODOs, FIXMEs, etc.)

## Code Style

- Must pass `golangci-lint` checks
- Follow existing code conventions
- Comments should be in English but clear and concise for non-native speakers

## Reporting Issues

- Use the GitHub Issues page
  - Issues may be in English, Spanish or Japanese
- For bugs, provide steps to reproduce
  - Providing an example code to reproduce is highly appreciated! That helps a lot!
- For feature requests, explain the use case and benefits
  - Providing an example code (`ExampleXXXXX` function) is highly appreciated! That helps a lot!
